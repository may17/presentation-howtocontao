Szenario 1:
Wir haben eine verschachtelte Navigationsstruktur und wollen das bei Klick auf die Oberseite direkt die erste Unterseite angesteuert wird. Im nachfolgenden Beispiel klick auf Über uns -> Standorte.

  • Über uns
    • Standorte
    • Team

Lösung 1
Wir setzen im Backend den Typ der Seite Über uns auf Interne Weiterleitung und weisen ihr die Referenzseite Standorte zu.

Was ist falsch an diesem Ansatz?
Verschieben wir die Seite Standorte an die zweite Position so muss auch die Verknüpfung neu gesetzt werden. Das ist eine potentielle fehlerquelle.

Bessere Lösung
Wir gehen wie in Lösung 2 vor jedoch weisen wir keine Referenzsseite zu. Contao springt dann direkt zur ersten Unterseite. Dies sagt einem auch der Hilfe text unter dem Referenzfeld.


Zusätzlicher Vorteil: Dupliziert man die gesamte Seitenstruktur z.B. für eine weitere Sprache so werden die Verknüpfungen nicht mit dupliziert.

Szenario 3:
Wir wollen eine Bilddatei über das Contao Backend auf einem Kundenhosting hochladen. Contao antwortet uns jedoch mit der Fehlermeldung Fehler 6 beim hochladen der Datei test.jpg

Frage:
Was will uns das sagen?

Fehler 6 bezieht sich auf den Fehlercode 6 beim Upload von Dateien via PHP. Der Fehlercode ist Bestandteil des $_FILES arrays.


Auf PHP.net findet man dazu folgenden Hinweis:
UPLOAD_ERR_NO_TMP_DIR
Value: 6; Missing a temporary folder. Introduced in PHP 4.3.10 and PHP 5.0.3.

Daraus ergibt sich das wir die php Konfigurationsvariable “upload_tmp_dir” genauer betrachten müssen. Z.b. mittels einer phpinfo Datei.


Steht dort no value so kann man bereits den Fehler eingrenzen und Kontakt zum Hoster aufnehmen. Alternativ gilt es die Schreibrechte des angegebenen Ordners überprüfen zu lassen.

Es gibt noch weitere Fehlercodes.


http://php.net/manual/en/features.file-upload.errors.php

Szenario 7:
Wir möchten ein anderes Slider-Script verwenden als swipe.js. Z.B. den BxSlider http://bxslider.com/

Lösung 1
Wir investieren die Zeit um eines der unzähligen Slidermodule für Contao herauszusuchen.

Lösung 2
Wir bauen uns einfach schnell eine eigene j_ Datei und binden diese Alternativ ein.

Schritt 1
Wir laden den bxslider runter und legen ihn unter files ab.

Schritt 2
Wir duplizieren die Datei j_slider.html in das templates Verzeichnis.

Schritt 3
Wir bearbeiten die Datei und binden das Script sowie die css Datei ein. 


            <?php

                // Add the swipe style sheet
                $GLOBALS['TL_CSS'][] = 'files/contaodemo/theme/js/bxslider/jquery.bxslider.css';
            ?>

            <script src="files/contaodemo/theme/js/bxslider/jquery.bxslider.min.js"></script>

Schritt 4
Wir implementieren das Script. 


            <script>
                (function($) {
                    $(document).ready(function() {
                        $('.ce_sliderStart').each(function(i, cte) {
                            var s = $('.content-slider', cte)[0],
                            c = s.getAttribute('data-config').split(',');

                            $('.slider-wrapper').bxSlider({

                                'autoStart': parseInt(c[0]),
                                'speed': parseInt(c[1]),
                                'startSlide': parseInt(c[2]),
                                'infiniteLoop': parseInt(c[3])
                            });
                        });
                    });
                })(jQuery);
            </script>

Schritt 5
Wir duplizieren die Template Datei ce_slider_stop.html5 und entfernen die Buttons.
